# 贡献与开发规范

## 分支

- 功能：`feat/<topic>`
- 修复：`fix/<topic>`
- 工程：`chore/<topic>`
- 文档：`docs/<topic>`

## 提交信息

采用 Conventional Commits：

- `feat:` 新功能
- `fix:` 缺陷修复
- `docs:` 文档
- `test:` 测试
- `refactor:` 重构
- `chore:` 工程与依赖

## Pull Request 要求

- 关联对应 Issue；
- 说明用户价值、实现方案和影响范围；
- 提供测试步骤与测试结果；
- 涉及 UI 时提供截图或录屏；
- 涉及数据模型时提供迁移和回滚说明；
- 不得提交密钥、生产数据或包含个人信息的样例文件。

## Definition of Done

- 验收标准全部满足；
- 类型检查、静态检查和测试通过；
- 新逻辑具备必要测试；
- 错误状态和空状态已处理；
- 文档和配置同步更新；
- 不破坏已有数据版本与仪表盘配置兼容性。
