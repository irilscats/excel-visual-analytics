# 数据刷新与版本设计

## 目标

用户保存分析、仪表盘和大屏后，更新同一业务数据时无需重新排版；系统重新执行原分析配置并展示最新结果。

## 保存内容

系统保存：

- 稳定字段 ID 与当前表头映射；
- 字段类型、角色、格式和默认聚合；
- 分析中的维度、指标、筛选、排序和公式；
- 图表样式、组件位置、尺寸和联动规则；
- 数据源当前活动版本。

系统不把截图作为可视化对象的唯一保存形式。

## 刷新流程

```text
上传或同步新版文件
→ 创建候选 DataVersion
→ 解析 Sheet 与字段
→ 执行字段差异检测
→ 校验类型、公式和依赖
→ 生成预览与影响报告
→ 校验通过后原子切换活动版本
→ 失效查询缓存
→ 重新计算分析和大屏数据
```

## 字段匹配优先级

1. 已保存的源字段标识；
2. 规范化后的表头完全匹配；
3. 用户确认过的历史别名；
4. 名称相似度、位置和数据类型综合推断；
5. 无法可靠匹配时要求人工重新映射。

## 结构变化策略

### 仅新增数据行

自动刷新，无需人工处理。

### 列顺序变化

按稳定字段 ID 和表头映射，不依赖列序号。

### 新增字段

保留所有现有图表，并提示新字段可用于分析。

### 字段改名

给出候选映射和置信度，用户确认后记录为历史别名。

### 字段删除

依赖该字段的图表进入 `needs-remap` 状态；保留布局和样式，其他图表继续工作。

### 字段类型变化

阻止自动发布候选版本，展示受影响的公式、筛选和图表。

## 失败保护

- 候选版本校验失败时不替换活动版本；
- 所有刷新任务具有幂等键；
- 记录新增行、删除行、字段变化和受影响对象；
- 用户可将任一成功版本切换为活动版本；
- 回滚数据版本不改变仪表盘布局配置。

## 更新方式规划

### MVP

- 手动上传新版 Excel/CSV；
- 页面打开时读取当前活动版本；
- 用户点击刷新重新执行查询。

### 后续

- 云盘文件定时同步；
- 服务器目录监听；
- 数据库与 API 定时刷新；
- WebSocket 或 Server-Sent Events 推送刷新状态。
