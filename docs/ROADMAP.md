# 开发路线图

## M0：工程基线

- Monorepo 目录与包管理；
- Web、API、Worker 最小应用；
- PostgreSQL、Redis、对象存储本地开发环境；
- 代码格式、静态检查、测试与 GitHub Actions；
- 统一错误响应、日志和环境变量规范。

**完成标准：** 新开发者可通过一条命令启动完整本地环境。

## M1：Excel/CSV 导入

- 文件上传与安全校验；
- Sheet 枚举和分页预览；
- 表头行与数据范围配置；
- 异步解析任务与进度；
- 数据版本和原始文件存储。

**完成标准：** 可稳定导入常规 Excel/CSV，并查看结构化预览。

## M2：字段模型与数据处理

- 字段类型与角色推断；
- 用户人工修正；
- 空值、重复值和格式概览；
- 字段重命名和基础计算字段；
- 字段稳定 ID 与历史别名。

**完成标准：** 不固定表头的文件可被转换为统一数据模型。

## M3：分析工作台

- 维度、指标和聚合；
- 筛选、排序与 Top N；
- 指标卡、柱状图、条形图、折线图、饼图、散点图和数据表；
- 保存、复制和再次编辑分析对象。

**完成标准：** 用户不写代码即可完成基础多维分析。

## M4：仪表盘与大屏

- 栅格仪表盘和自由画布；
- 拖拽、缩放、图层、锁定与复制；
- 添加已保存分析；
- 全局筛选和基础联动；
- 保存、预览和全屏展示。

**完成标准：** 分析结果可复用并形成可保存的业务看板。

## M5：刷新、差异与回滚

- 新版数据上传；
- 字段差异检测和影响分析；
- 活动版本原子切换；
- 查询缓存失效和图表刷新；
- 重新映射与版本回滚。

**完成标准：** 同结构文件更新后，大屏数据更新而布局保持不变。

## M6：MVP 加固与发布

- 大文件和并发性能测试；
- 权限和上传安全测试；
- 错误恢复和任务幂等测试；
- 演示数据与端到端测试；
- 部署文档、备份策略和发布检查表。

**完成标准：** MVP 可部署、可演示、可恢复并具备基础运维能力。
